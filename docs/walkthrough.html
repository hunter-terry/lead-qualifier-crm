How it actually works — no code required

What happens when a lead lands in your CRM

This walks through lead-qualifier-crm, a real automation that scores incoming sales leads with AI and writes the result straight into a real CRM — Airtable — instead of a file on a hard drive. Both examples below are real runs against a live Airtable base, executed live, not mocked up.

The pipeline

Five steps — the same shape whether the destination is Airtable, HubSpot, or any other CRM.

1
Lead arrives
A new inquiry lands — from a web form, an export, however it currently reaches you.
2
AI reads it
A local model reads the message and rates intent, urgency, and a 1–10 fit score.
3
Checked, not trusted
A plain rule check confirms the AI's answer is actually usable — before anything moves forward.
4
Written to the CRM
The result lands as a real record in Airtable — visible, searchable, ready for a human to act on.

This is the actual point of the exercise. The lead on the right had no message at all — and the AI still went ahead and scored it, same as it did for a real lead, based on whatever scraps it had. Left unchecked, that's a blank contact silently entering the CRM as if it were a real prospect. The rule-based check is what caught it — not the AI noticing its own mistake, because it didn't. And unlike simply discarding bad leads, this design still writes them to Airtable marked invalid, so nothing that came in through the form disappears without a trace.
